# Phase 11: Proactive Support & Feedback Loops Plan

<objective>
Build a data-driven support ecosystem that anticipate user needs and creates a continuous improvement loop through active feedback collection.
</objective>

<tasks>

### Task 1: Proactive Alerting UI
- **Health Banners**: Display global banners in the dashboard if background workers (Scraper/Enrichment) are experiencing high latency.
- **Predictive Support**: If a user's scrape returns 0 results, show an automated "Troubleshooting Tip" about niche keywords and location filtering.

### Task 2: Multi-Channel In-App Support
- **Intercom/Crisp Integration**: Embed a live-chat widget for real-time support.
- **Support Sidebar**: Add a "Need Help?" floating button that opens a mini-KB (Knowledge Base) with search.

### Task 3: Feedback & Roadmap Integration
- **NPS / CSAT Collection**: Trigger a 1-question survey ("How would you rate this batch's quality?") after a user views an AI-enriched batch.
- **Upvote Portal**: Create a "Request Feature" page where users can suggest niches or template styles and upvote others' ideas.

</tasks>

<success_criteria>
- Support ticket response time is reduced through in-app auto-resolution tips.
- Roadmap prioritization is backed by user vote data.
</success_criteria>
